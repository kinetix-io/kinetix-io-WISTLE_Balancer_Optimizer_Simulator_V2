WISTLE: Work Instruction & Standard Time = Line Efficiency

Comprehensive Enterprise Manufacturing Optimization Engine | Version 16.1

1. Introduction & Core Objective

WISTLE is an advanced, algorithm-driven enterprise simulation platform custom-engineered for LG manufacturing lines (AC, RF, WM, TV). The platform’s core objective is to allow Line Optimization Managers to ingest raw cycle times (TAKTDATA) and GERP master standards, dynamically visualize line bottlenecks, and simulate the exact impact of micro-task routing and manpower reallocation without disrupting live shop-floor operations.

2. Dashboard Module (Page 1)

The Dashboard serves as the data ingestion and high-level verification center.

  • File Uploads & Validation: Users drop standard CSV matrices into the respective target zones. The system actively enforces a strict prefix rule matching your selected Business Unit (e.g., files must begin with AC_ for Air Conditioners). Visual cues (Green for Success, Red for Fail) indicate ingestion integrity.
  • Refresh Data (↻): Instantly recalculates all global arrays and states in case underlying backend structures or target metrics are updated by the user.
  • Model Performance Ledger: A grid isolating distinct product families to monitor how efficiently a specific model runs against its localized GERP Standard Time based on currently allocated MP.
  • Stage Shuffling Overrides (±20 Matrix): Allows the user to constrain how the Auto-Optimizer traverses the line. By default, the engine only looks at immediate neighbors ($\pm1$ step). However, users can type explicit bounds like `5-20` to restrict micro-task migration strictly between stage 5 and stage 20. Includes CSV Export/Import for saving operational templates.
  • Raw Data Viewer: A collapsible audit view showing the first 25 rows of the uploaded CSVs to visually confirm data integrity.

Dashboard KPIs Explained

  • Tasks, Stages, Models: Count of unique components mapped from the arrays.
  • Total MP: The absolute sum of all operator headcount ($\Sigma MP$) designated across active layout stages.
  • Total Work Content ($\Sigma WC$): The arithmetic sum of all manual processing time allocated across the entire line (in minutes).
  • Overload Stages: The physical count of stations currently exceeding the user's defined "STD Takt Benchmark" target.
  • Max Bottleneck ($T_{max}$): Identifies the slowest individual station pacing the entire line. (Station Work Content / Station MP).
  • Yield Rate (Est.): Operational throughput assumption factor.
  • UPH (Units Per Hour): Estimated hourly output rate. Formula: 60 / Max Bottleneck.
  • UPPH (Units Per Person Hour): Estimated output per operator. Formula: UPH / Total MP.
  • Efficiency (LBE %): The global line balancing efficiency score.
  • Balance Delay: The wasted idle capacity percentile across the line. Formula: 100 - Line Efficiency %.

3. Optimizer Module (Page 2)

The Optimizer is the interactive simulation matrix where the algorithm and manual overrides interact to resolve bottlenecks.

  • Filter Toolbars: Multi-select dropdowns isolate specific manufacturing tool groups or localized models. The `STD Takt Benchmark` input defines the line threshold; exceeding this target marks a stage as "Breached".
  • BREACHED (Graph Only) Toggle: Hides perfectly balanced stages in the visualizer graph, bringing immediate focus only to stages marked in red (exceeding benchmark). The tabular data remains unfiltered so spatial routing context is preserved.
  • Unified Zoom (⛶): Expands the Graph and Table into a full-screen, distraction-free environment for intensive data analysis.
  • Interactive Bar Chart: Renders micro-tasks as stacked layers. Safe stages render as a gradient of Green to Yellow. If the cumulative micro-task time breaks the Target Benchmark, the breaching blocks render as Translucent Red. Hover over bars to read specific task labels (e.g., `S1_Pick the LED`).
  • Tabular Grid & Manpower Update: The grid lays out explicit operations. Manpower (MP) is controlled via direct text-input fields or +/- buttons. Adjusting these recalculates the entire line topology instantly.
  • Auto-Optimize Algorithm (Cascade Fallback): A heuristic engine that evaluates the line and sorts all breaching bottlenecks from worst to least. It attempts to migrate tasks from the worst bottleneck to adjacent valid stations. If a bottleneck is constrained and completely stuck, the engine automatically falls back to the next worst bottleneck in the list (e.g., Stage 11) to ensure optimization continues relentlessly. Crucial Constraint: The engine ONLY shuffles micro-tasks; it is mathematically barred from altering Manpower allocations independently.

4. Audit Report Module (Page 3)

Transparency is paramount for line approvals. Every action taken in the Optimizer is logged explicitly into the exhaustive Iteration Ledger.

  • Iter #: Tracks discrete optimization events. "Auto-Optimize" actions that take multiple steps are batched under a unified Iteration ID.
  • Source / Target Metrics: The ledger captures the precise Stage Number, Stage Name, Station Manpower, and the exact Takt shift (Old → New) for both the sending and receiving stations.
  • Global Line Impact: Tracks the overarching Line Bottleneck, the before-and-after Global LBE, and the Delta shift per micro-move.
  • Export Controls: Users can download individual iteration steps or the entire master history as a CSV for management review.

5. Mathematical Core Formulations

The WISTLE engine strictly adheres to standard manufacturing engineering formulas for all internal states.

Station Work Content ($WC_s$) = $\sum$ (Micro-Task Cycle Times at Station)

Station Takt ($T_s$) = $WC_s$ / Station Manpower ($MP_s$)

Line Bottleneck ($T_{max}$) = $\max(T_s)$ across all active stations

Line Balancing Efficiency (LBE %) = [ $\sum WC_s$ ] / [ $\sum MP_s$ × $T_{max}$ ] × 100

Worked Calculation Example:

  • Line Work Content ($\Sigma WC$) = 2.4643 mins
  • Line Manpower ($\Sigma MP$) = 8 Operators
  • Governing Bottleneck ($T_{max}$) = Stage 9 (0.4375 mins)
  • LBE = 2.4643 / (8 × 0.4375) × 100 = 70.41% Efficiency
